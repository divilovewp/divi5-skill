## 1. Overview & Purpose

### What is DIVI5 JSON?

DIVI5 is a WordPress page builder by Elegant Themes. In DIVI5, every page is stored as WordPress Block Editor (Gutenberg) markup embedded inside a JSON file. The JSON file is imported into a WordPress site using DIVI5's import tool, instantly recreating the full layout.

The content inside the JSON is a string of HTML-style WordPress block comments — each comment represents a DIVI5 module, row, column, or section.

### How JSON Files Are Deployed

1. Export a JSON from DIVI5's builder (or generate one programmatically).
2. In the WordPress admin, go to DIVI → Import & Export (or use the DIVI5 builder's import button).
3. Upload the JSON file. The builder parses the `data` key and renders the page.

### Key Principles — Always Follow These

- **Every page JSON wraps content in `<!-- wp:divi/placeholder -->...<!-- /wp:divi/placeholder -->`**.
- **The hierarchy is always: Section → Row → Column → Module(s)**. Never skip levels.
- **All module attributes are JSON-encoded inside the HTML comment**, using `\"` escape sequences for quotes within the attribute string.
- **Self-closing modules** use `/>` at the end: `<!-- wp:divi/heading {...} /-->`.
- **Container modules** (section, row, column, group, accordion, etc.) use open/close pairs: `<!-- wp:divi/section {...} -->` ... `<!-- /wp:divi/section -->`.
- **`builderVersion`** must be present on every module. Always use `"5.0.1"` (current latest version).
- **Responsive values** are specified per breakpoint: `desktop`, `tablet`, `phoneWide`, `phone`.
- **Global color and variable tokens** are referenced via the `$variable(...)$` syntax (see Section 5).
- **Never include `groupPreset`** in generated JSON — it references site-specific preset IDs that won't exist on other installations.
- **Spacing is optional** — DIVI5 applies sensible default padding/margin when none is specified. Only add explicit spacing when overriding defaults.
- **Always use placeholder images** (base64 SVG) unless the user provides real image URLs.
- **Always ask for brand colors** before generating JSON for a new site — never assume or default to another site's color tokens.
- **Only include decorative Code-module backgrounds** (e.g. animated orbs) when the design explicitly calls for them — keep generic page JSON free of site-specific decorative CSS.

## 2. Top-Level JSON Structure

Every valid DIVI5 page JSON has this root structure:

```json
{
  "context": "et_builder",
  "data": {
    "POST_ID": "<!-- wp:divi/placeholder -->...PAGE_CONTENT...<!-- /wp:divi/placeholder -->"
  },
  "presets": null,
  "global_colors": [],
  "global_variables": [],
  "canvases": {
    "local": [],
    "global": []
  },
  "images": [],
  "thumbnails": []
}
```

### Field Descriptions

| Key | Required | Description |
|-----|----------|-------------|
| `context` | Yes | Always `"et_builder"` |
| `data` | Yes | Object with one key (any integer string, e.g. `"11"`) whose value is the full page markup string |
| `presets` | No | `null` for pages without saved presets |
| `global_colors` | No | Array of `[id, {color, status, label}]` tuples — only present when exporting site-wide design variables |
| `global_variables` | No | Array of variable objects — same use case |
| `canvases` | No | Off-page canvas definitions — usually `{"local":[],"global":[]}` |
| `images` | No | Empty array `[]` for most exports |
| `thumbnails` | No | Empty array `[]` for most exports |

### Minimal Working Skeleton

```json
{
  "context": "et_builder",
  "data": {
    "11": "<!-- wp:divi/placeholder --><!-- wp:divi/section {\"builderVersion\":\"5.0.1\"} -->\r\n<!-- wp:divi/row {\"module\":{\"advanced\":{\"flexColumnStructure\":{\"desktop\":{\"value\":\"equal-columns_1\"}}},\"decoration\":{\"layout\":{\"desktop\":{\"value\":{\"flexWrap\":\"nowrap\"}}}}},\"builderVersion\":\"5.0.1\"} -->\r\n<!-- wp:divi/column {\"module\":{\"decoration\":{\"sizing\":{\"desktop\":{\"value\":{\"flexType\":\"24_24\"}}}}},\"builderVersion\":\"5.0.1\"} -->\r\n<!-- wp:divi/text {\"content\":{\"innerContent\":{\"desktop\":{\"value\":\"\\u003cp\\u003eHello World\\u003c/p\\u003e\"}}},\"builderVersion\":\"5.0.1\"} /-->\r\n<!-- /wp:divi/column -->\r\n<!-- /wp:divi/row -->\r\n<!-- /wp:divi/section --><!-- /wp:divi/placeholder -->"
  },
  "presets": null,
  "global_colors": [],
  "global_variables": [],
  "canvases": {"local": [], "global": []},
  "images": [],
  "thumbnails": []
}
```

### Important Encoding Rules

Inside the `data` string:
- Line breaks are `\r\n`
- All inner JSON attribute quotes are escaped: `\"` becomes `\\"` inside the outer JSON string
- HTML entities are unicode-escaped: `<` becomes `\\u003c`, `>` becomes `\\u003e`, `/` becomes `\\/`
- The entire page content string must be valid when parsed (double-decode step)

## 3. Module Reference

### 3.1 Layout Modules (Structural)

These modules define the page structure and are not content themselves.

---

#### `divi/section`

The top-level container for all page content. Every page must have at least one section.

**Self-closing?** No — uses open/close tags.

**Key attributes:**
```json
{
  "module": {
    "meta": {
      "adminLabel": {"desktop": {"value": "Hero Section"}}
    },
    "decoration": {
      "layout": {"desktop": {"value": {"justifyContent": "center", "display": "block"}}},
      "background": {
        "desktop": {
          "value": {
            "color": "#0f172a",
            "gradient": {
              "enabled": "on",
              "type": "linear",
              "stops": [
                {"position": 0, "color": "#0f172a"},
                {"position": 100, "color": "#1e293b"}
              ]
            }
          }
        }
      },
      "spacing": {
        "desktop": {"value": {"padding": {"top": "80px", "bottom": "80px", "syncVertical": "off", "syncHorizontal": "off"}}}
      }
    }
  },
  "builderVersion": "5.0.1"
}
```

**Notes:**
- `adminLabel` helps identify sections in the builder UI.
- Omit `spacing` entirely to use DIVI5 default padding — only add it when overriding defaults.
- Never include `groupPreset` in generated JSON — preset IDs are site-specific.

---

#### `divi/row`

A horizontal container inside a section. Defines how columns are arranged.

**Self-closing?** No.

**Key attributes:**
```json
{
  "module": {
    "advanced": {
      "flexColumnStructure": {
        "desktop": {"value": "equal-columns_2"}
      }
    },
    "decoration": {
      "layout": {
        "desktop": {"value": {"flexWrap": "nowrap"}},
        "phone": {"value": {"flexWrap": "wrap"}},
        "phoneWide": {"value": {"flexWrap": "wrap"}}
      },
      "sizing": {
        "desktop": {"value": {"maxWidth": "1280px"}}
      }
    }
  },
  "builderVersion": "5.0.1"
}
```

**`flexColumnStructure` values:**

| Value | Columns |
|-------|---------|
| `"equal-columns_1"` | 1 column (full width) |
| `"equal-columns_2"` | 2 equal columns |
| `"equal-columns_3"` | 3 equal columns |
| `"equal-columns_4"` | 4 equal columns |

**Notes:**
- On mobile (`phone`, `phoneWide`), always set `"flexWrap": "wrap"` to stack columns.
- `maxWidth` on the row controls the content container width.

---

#### `divi/column`

A column inside a row. Contains content modules.

**Self-closing?** No.

**Key attributes:**
```json
{
  "module": {
    "decoration": {
      "sizing": {
        "desktop": {"value": {"flexType": "12_24"}},
        "phone": {"value": {"flexType": "24_24"}},
        "phoneWide": {"value": {"flexType": "24_24"}}
      },
      "layout": {
        "desktop": {"value": {"justifyContent": "center"}}
      },
      "background": {
        "desktop": {
          "value": {
            "color": "$variable({\"type\":\"color\",\"value\":{\"name\":\"gcid-h0ozdk0ylc\",\"settings\":{}}})$"
          }
        }
      },
      "border": {
        "desktop": {
          "value": {
            "styles": {"all": {"width": "1px", "color": "$variable({\"type\":\"color\",\"value\":{\"name\":\"gcid-c0jucvsoon\",\"settings\":{}}})$"}},
            "radius": {"topLeft": "2rem", "topRight": "2rem", "bottomLeft": "2rem", "bottomRight": "2rem", "sync": "on"}
          }
        }
      }
    },
    "advanced": {
      "type": {"desktop": {"value": "4_4"}}
    }
  },
  "builderVersion": "5.0.1"
}
```

**`flexType` values (column width as fraction of 24):**

| flexType | Width |
|----------|-------|
| `"24_24"` | 100% (full width) |
| `"12_24"` | 50% (half) |
| `"8_24"` | 33.3% (one-third) |
| `"6_24"` | 25% (one-quarter) |
| `"16_24"` | 66.6% (two-thirds) |
| `"18_24"` | 75% |

---

#### `divi/group`

A generic container module that can hold any other module inside a column. Think of it as a `div` wrapper for grouping content.

**Self-closing?** No.

**Usage:**
```
Column → Group → [Text, Button, Image, etc.]
```

**Minimal example:**
```json
{"builderVersion": "5.0.1"}
```

---

#### `divi/group-carousel`

Like `divi/group`, but displays its children `divi/group` instances as carousel slides.

**Self-closing?** No.

**Structure:**
```
Section → Row → Column → group-carousel → group (slide 1) → [modules]
                                        → group (slide 2) → [modules]
```

#### `divi/text`

Renders a block of HTML content (paragraphs, lists, etc.).

**Self-closing?** Yes (`/-->`).

**Key attributes:**
```json
{
  "content": {
    "innerContent": {
      "desktop": {"value": "<p>Your text here.</p>"}
    },
    "decoration": {
      "bodyFont": {
        "body": {
          "font": {
            "desktop": {
              "value": {
                "size": "$variable({\"type\":\"content\",\"value\":{\"name\":\"gvid-6vg4l84jp4\",\"settings\":{}}})$",
                "family": "$variable({\"type\":\"content\",\"value\":{\"name\":\"--et_global_body_font\",\"settings\":{}}})$",
                "color": "$variable({\"type\":\"color\",\"value\":{\"name\":\"gcid-body-color\",\"settings\":{}}})$",
                "style": ["uppercase"],
                "letterSpacing": "2px"
              }
            }
          }
        }
      },
      "headingFont": {
        "h1": {
          "font": {
            "desktop": {
              "value": {
                "size": "$variable({\"type\":\"content\",\"value\":{\"name\":\"gvid-1ndmhhes58\",\"settings\":{}}})$",
                "family": "$variable({\"type\":\"content\",\"value\":{\"name\":\"--et_global_heading_font\",\"settings\":{}}})$",
                "weight": "700",
                "lineHeight": "$variable({\"type\":\"content\",\"value\":{\"name\":\"gvid-7q5xf21pob\",\"settings\":{}}})$",
                "textAlign": "center"
              }
            }
          }
        }
      }
    }
  },
  "module": {
    "decoration": {
      "layout": {"desktop": {"value": {"justifyContent": "center", "alignItems": "center"}}},
      "spacing": {"desktop": {"value": {"padding": {"syncVertical": "on", "syncHorizontal": "off"}}}}
    }
  },
  "builderVersion": "5.0.1"
}
```

**Note:** Text content values must HTML-encode `<` as `<` and `>` as `>`.

## 4. Nesting Rules & Hierarchy

### 4.1 The Fundamental Hierarchy

```
Section
  └── Row
        └── Column
              └── Module(s) or Group
                    └── Module(s) [if inside Group]
```

**Rule:** You CANNOT place a module directly inside a Section. It must go Section → Row → Column → Module.

### 4.2 Nesting a Row Inside a Column

DIVI5 supports nested rows — a Row can be placed inside a Column (which itself is inside an outer Row). This allows complex grid layouts.

```
Section
  └── Outer Row (2 columns)
        └── Left Column
        │     └── Group
        │           └── Image
        │           └── Heading
        │           └── Button
        └── Right Column
              └── Inner Row (1 column)
                    └── Column
                          └── Blurb
```

### 4.3 Group Nesting

A `divi/group` can contain any module. It can also contain a nested row structure:

```
Section
  └── Row (1 column)
        └── Column
              └── Group
                    └── Row (2 columns)   ← Row INSIDE a Group
                          └── Column → Heading
                          └── Column → Text
                    └── Text Module       ← Module after nested row in same group
```

### 4.4 JSON Representation of Nesting

Nesting is represented by the order of block comment opening and closing tags. A child appears between the parent's opening comment and closing comment.

**Example: Section → Row → Column → Text**

```
<!-- wp:divi/section {"builderVersion":"5.0.1"} -->
  <!-- wp:divi/row {"module":{...},"builderVersion":"5.0.1"} -->
    <!-- wp:divi/column {"module":{...},"builderVersion":"5.0.1"} -->
      <!-- wp:divi/text {"content":{...},"builderVersion":"5.0.1"} /-->
    <!-- /wp:divi/column -->
  <!-- /wp:divi/row -->
<!-- /wp:divi/section -->
```

**Example: Group inside Column with multiple modules:**

```
<!-- wp:divi/column {...} -->
  <!-- wp:divi/group {"builderVersion":"5.0.1"} -->
    <!-- wp:divi/heading {...} /-->
    <!-- wp:divi/text {...} /-->
    <!-- wp:divi/button {...} /-->
  <!-- /wp:divi/group -->
<!-- /wp:divi/column -->
```

### 4.5 Nesting Rules Summary

| Parent | Can Contain |
|--------|-------------|
| `section` | `row`, `global-layout` |
| `row` | `column` |
| `column` | `group`, `group-carousel`, any module, nested `row` |
| `group` | any module, nested `row` |
| `group-carousel` | `group` (each group = one slide) |
| `accordion` | `accordion-item` |
| `counters` | `counter` |
| `contact-form` | `contact-field` |
| `pricing-tables` | `pricing-table` |
| `slider` | `slide` |
| `tabs` | `tab` |
| `social-media-follow` | `social-media-follow-network` |
| `icon-list` | `icon-list-item` |
| `video-slider` | `video-slider-item` |

### 4.6 Nesting Mistakes to Avoid

- ❌ Module directly in a Section (no Row/Column)
- ❌ Module directly in a Row (no Column)
- ❌ Nested Section inside a Column (sections cannot nest)
- ❌ `accordion-item` outside an `accordion`
- ❌ `slide` outside a `slider`
- ❌ Missing closing tag `<!-- /wp:divi/section -->` for any non-self-closing module
- ❌ `flexColumnStructure` count mismatching actual column count

### 4.7 `flexColumnStructure` vs. Actual Columns

When a row has `"equal-columns_3"` in `flexColumnStructure`, it must contain exactly 3 `divi/column` children. If column count doesn't match the structure declaration, DIVI may render incorrectly.

## 5. DIVI5 Variables / Design Tokens

### 5.1 What Are Variables?

DIVI5 supports a global design system with two types of tokens:
- **Global Colors** (`global_colors`) — named color values (hex, rgba)
- **Global Variables** (`global_variables`) — any CSS value, commonly font sizes using `clamp()`

These are defined in the site's design settings and can be referenced inside any module's style properties.

### 5.2 Variable Reference Syntax

To use a variable inside a module attribute value, use the `$variable(...)$` syntax:

**For colors:**
```
$variable({"type":"color","value":{"name":"gcid-VARIABLE_ID","settings":{}}})$
```

**For numeric/font variables:**
```
$variable({"type":"content","value":{"name":"gvid-VARIABLE_ID","settings":{}}})$
```

**For font family variables (global fonts use CSS custom property names):**
```
$variable({"type":"content","value":{"name":"--et_global_heading_font","settings":{}}})$
$variable({"type":"content","value":{"name":"--et_global_body_font","settings":{}}})$
```

### 5.4 How to Reference Variables in Module Code

**Using a color in background:**
```json
"background": {
  "desktop": {
    "value": {
      "color": "$variable({\"type\":\"color\",\"value\":{\"name\":\"gcid-primary-color\",\"settings\":{}}})$"
    }
  }
}
```

**Using a font size variable:**
```json
"size": "$variable({\"type\":\"content\",\"value\":{\"name\":\"gvid-qgdrs7tgxr\",\"settings\":{}}})$"
```

**Using a global font variable:**
```json
"family": "$variable({\"type\":\"content\",\"value\":{\"name\":\"--et_global_heading_font\",\"settings\":{}}})$"
```

**Important:** Inside the JSON data string, these variable references will be double-escaped when the outer JSON is serialized. Write them as shown above in the module attribute object; the outer JSON serializer handles the escaping.

When you want to include global tokens in an export file (e.g., for a complete site export), they go in the root-level `global_colors` and `global_variables` arrays.


### 6.3 Responsive Breakpoints

DIVI5 uses four breakpoints:

| Key | Screen Size |
|-----|-------------|
| `desktop` | 981px and above (required base) |
| `tablet` | 768px–980px |
| `phoneWide` | 480px–767px |
| `phone` | Below 480px |

**Rule:** Always provide `desktop` values. Add `tablet`, `phoneWide`, and `phone` only where responsive behavior differs from desktop.

**Common responsive patterns:**
- Row `flexWrap`: `"nowrap"` on desktop → `"wrap"` on `phoneWide`/`phone`
- Column `flexType`: `"12_24"` on desktop → `"24_24"` on `phone`
- Font sizes: use `clamp()` variables to auto-scale
- Padding: reduce on smaller screens

## 7. Real-World Patterns

### 7.1 Global Layout (Repeated Global Module)

A site may use a `divi/global-layout` at the top of each section to inject a shared row (for example, a decorative animated background). A global layout references a saved global module by id and is injected like this:

```
<!-- wp:divi/global-layout {"globalModule":"209","blockName":"divi/row","localAttrs":{...}} -->
  <!-- wp:divi/column {...,"globalParent":"209"} -->
    <!-- wp:divi/code {...code module...} /-->
    <!-- wp:divi/code {...style module...} /-->
  <!-- /wp:divi/column -->
<!-- /wp:divi/global-layout -->
```

The code modules inject HTML for decorative elements and a `<style>` block, creating a shared visual effect across every page that references the global layout.

### 7.2 Hero Section Pattern

A common hero pattern used across marketing pages.

**Structure:** 2-column row (text left, image right), reverses on mobile.

```
Section (Hero, dark bg, full-height)
  └── Global Layout (optional shared-background row)
  └── Row (2-col, max-width 1280px, reverse on mobile)
        └── Left Column (12/24 → 24/24 mobile, justify center)
        │     └── Text (eyebrow/label, uppercase, accent color)
        │     └── Text (h1 heading, big heading variable)
        │     └── Text (subheading, body color)
        │     └── Button (primary color, box shadow)
        └── Right Column (12/24 → 24/24 mobile)
              └── Image (rounded, hero image)
```

**Key section attributes:**
```json

      "layout": {"desktop": {"value": {"justifyContent": "center"}}},

    }
  },
  "builderVersion": "5.0.1"
}
```

## 8. Step-by-Step: How to Build a Page JSON from Scratch

### Step 1: Understand the Page Brief

Before writing any JSON, answer these questions:
- What is the page's primary purpose? (e.g., services page, about page, landing page)
- What sections are needed? (e.g., hero, features, testimonials, CTA, contact form)
- What is the color scheme and typography? (Use existing global tokens if available)
- Does this page belong to an existing site with shared design tokens?

### Step 2: Plan the Layout Structure

Map out each section before coding:

```
[Section 1: Hero]
  - 2-column row: left=text+button, right=image
  - Dark bg with optional shared-background injection

[Section 2: Services]
  - 1-col row: label + heading
  - 3-col row: 3 service cards (glassmorphism)

[Section 3: CTA]
  - 1-col row: heading + text + button (centered)
```

### Step 3: Select Modules Based on Content

Use this decision guide:
- Rich text paragraph → `divi/text`
- Single heading → `divi/heading`
- Button only → `divi/button`
- Image only → `divi/image`
- Icon + title + description → `divi/blurb`
- Title + body + button combined → `divi/cta`
- Quote with attribution → `divi/testimonial`
- Multiple expand/collapse items → `divi/accordion`
- FAQ single item → `divi/toggle`
- Custom HTML/CSS → `divi/code`
- Multiple items grouped → `divi/group` + child modules
- Carousel of grouped content → `divi/group-carousel`

### Step 4: Build Each Section

For each section, write the block structure in order:

1. Open `<!-- wp:divi/section {...} -->`
2. Add a shared-background row if the site uses one (as global-layout)
3. For each row in the section:
   - Open `<!-- wp:divi/row {...} -->`
   - For each column:
     - Open `<!-- wp:divi/column {...} -->`
     - Add modules (with correct attributes and self-closing syntax)
     - Close `<!-- /wp:divi/column -->`
   - Close `<!-- /wp:divi/row -->`
4. Close `<!-- /wp:divi/section -->`

### Step 5: Apply Variables for Brand Consistency

**Before writing any color values, ask the user for:**
- Primary background color
- Heading text color
- Body text color
- Accent/highlight color
- Button color (and hover state)
- Any secondary colors used for cards, borders, etc.

Only proceed with hardcoded placeholder colors (e.g. `#0f172a`) if the user explicitly says they'll update them later.

Replace hardcoded values with variable references when the site has global tokens configured:
- Colors → `$variable({"type":"color","value":{"name":"gcid-XXX","settings":{}}})$`
- Font sizes → `$variable({"type":"content","value":{"name":"gvid-XXX","settings":{}}})$`
- Font families → `$variable({"type":"content","value":{"name":"--et_global_heading_font","settings":{}}})$`

### Step 6: Encode the Content String

When building the final JSON:
1. Write the page markup in readable form
2. HTML-encode: `<` → `<`, `>` → `>`, `/` → `\/` (in HTML values)
3. Escape inner JSON quotes: `"` → `\"`
4. Collapse to a single string value for the `data` key
5. Use `\r\n` for line breaks within the string

### Step 7: Wrap in Root JSON

```json
{
  "context": "et_builder",
  "data": {
    "11": "<!-- wp:divi/placeholder -->PAGE_CONTENT_HERE<!-- /wp:divi/placeholder -->"
  },
  "presets": null,
  "global_colors": [],
  "global_variables": [],
  "canvases": {"local": [], "global": []},
  "images": [],
  "thumbnails": []
}
```

If site-wide design tokens should travel with the file, populate `global_colors` and `global_variables`.

### Step 8: Validation Checklist

Before finalizing the JSON, verify:

- [ ] Root has `"context": "et_builder"`
- [ ] `data` key has exactly one entry (any integer string key)
- [ ] Page content starts with `<!-- wp:divi/placeholder -->` and ends with `<!-- /wp:divi/placeholder -->`
- [ ] Every section has at least one row; every row has at least one column
- [ ] Every non-self-closing module has a matching closing comment `<!-- /wp:divi/MODULE -->`
- [ ] Every self-closing module ends with ` /-->` (note the space before `/`)
- [ ] `builderVersion` is present on every module
- [ ] `flexColumnStructure` count matches actual column count in that row
- [ ] Column `flexType` values add up to `24_24` per row (or at minimum are consistent with the structure)
- [ ] All `$variable(...)$` references use valid IDs from the token list
- [ ] HTML content values properly encode `<` and `>`
- [ ] Mobile breakpoints provided for column widths and row wrapping

## 9. Common Mistakes & How to Avoid Them

### 9.1 Missing Required Fields

❌ **Problem:** Module without `builderVersion`
```json
<!-- wp:divi/text {"content":{"innerContent":{"desktop":{"value":"Hello"}}}} /-->
```

✅ **Fix:** Always include `"builderVersion":"5.0.1"`:
```json
<!-- wp:divi/text {"content":{"innerContent":{"desktop":{"value":"Hello"}}},"builderVersion":"5.0.1"} /-->
```

---

### 9.2 Invalid Nesting

❌ **Problem:** Placing a module directly in a section
```
Section → Text Module   ← INVALID
```

✅ **Fix:** Always use Section → Row → Column → Module

---

### 9.3 Mismatched flexColumnStructure

❌ **Problem:**
```json
// Row says 3 columns but only has 2 column tags
"flexColumnStructure": {"desktop": {"value": "equal-columns_3"}}
// But only 2 <!-- wp:divi/column --> present
```

✅ **Fix:** Count your columns and match them to `flexColumnStructure`.

---

### 9.4 Incorrect Variable Reference Syntax

❌ **Problem:** Missing quote escapes or wrong type
```json
"color": "$variable({"type":"color","value":{"name":"gcid-primary-color"}})"
```

✅ **Fix:** Use proper escaped JSON inside the string (when inside a JSON string context):
```json
"color": "$variable({\"type\":\"color\",\"value\":{\"name\":\"gcid-primary-color\",\"settings\":{}}})$"
```

---

### 9.5 Unescaped HTML in Content Values

❌ **Problem:**
```json
"value": "<p>Hello World</p>"
```

✅ **Fix:**
```json
"value": "<p>Hello World</p>"
```

---

### 9.6 Wrong Self-Closing Syntax

❌ **Problem:** Using `-->` instead of ` /-->`
```
<!-- wp:divi/text {"content":{...}} -->
```

✅ **Fix:** Self-closing modules must end with ` /-->` (space + slash before `-->`):
```
<!-- wp:divi/text {"content":{...}} /-->
```

---

### 9.7 Missing Closing Tags

❌ **Problem:** Forgetting to close a section/row/column/group
```
<!-- wp:divi/section {...} -->
  <!-- wp:divi/row {...} -->
    <!-- wp:divi/column {...} -->
  <!-- /wp:divi/row -->
<!-- /wp:divi/section -->
← Missing /wp:divi/column closing tag!
```

✅ **Fix:** Every open tag (section, row, column, group, accordion, etc.) needs a matching close tag.

---

### 9.8 Wrong Module Tag Names

❌ **Problem:** Using Divi 4 module names or guessing names
```
<!-- wp:divi/et_pb_text {...} -->
```

✅ **Fix:** Use the correct DIVI5 tag names:

| Wrong (DIVI4/guessed) | Correct (DIVI5) |
|----------------------|-----------------|
| `et_pb_text` | `divi/text` |
| `et_pb_image` | `divi/image` |
| `et_pb_button` | `divi/button` |
| `et_pb_column` | `divi/column` |
| `et_pb_row` | `divi/row` |
| `divi/person` | `divi/team-member` |

---

### 9.9 Missing `settings:{}` in Variable References

❌ **Problem:**
```json
"$variable({\"type\":\"color\",\"value\":{\"name\":\"gcid-primary-color\"}})$"
```

✅ **Fix:** Always include `"settings":{}`:
```json
"$variable({\"type\":\"color\",\"value\":{\"name\":\"gcid-primary-color\",\"settings\":{}}})$"
```

---

### 9.10 Not Adding Mobile Breakpoints

❌ **Problem:** Only desktop values, columns don't stack on mobile
```json
"sizing": {"desktop": {"value": {"flexType": "12_24"}}}
```

✅ **Fix:** Add phone breakpoints for columns:
```json
"sizing": {
  "desktop": {"value": {"flexType": "12_24"}},
  "phone": {"value": {"flexType": "24_24"}},
  "phoneWide": {"value": {"flexType": "24_24"}}
}
```
