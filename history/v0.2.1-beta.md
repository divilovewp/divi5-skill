## 1. Non-Negotiable Rules

Always follow these before writing any JSON:

1. **Every page content string is wrapped in `<!-- wp:divi/placeholder -->...<!-- /wp:divi/placeholder -->`.**
2. **Base hierarchy is: Section → Row → Column → Module(s).** Never skip levels. However, rows CAN nest inside columns (see rule 14).
3. **`builderVersion: "5.0.1"` must be on every single module** — section, row, column, and all content modules.
4. **Self-closing modules** end with ` /-->` (space + slash): `<!-- wp:divi/heading {...} /-->`.
5. **Container modules** (section, row, column, group, etc.) use open + close pairs.
6. **Row flex layout is driven by `flexColumnStructure` + `layout.*.value.flexWrap`** — NOT by a `display: flex` property. Divi applies flex automatically.
7. **Column widths are set via `module.decoration.sizing.*.value.flexType`** using the `N_24` fraction system.
8. **`flexColumnStructure` count must exactly match the number of `divi/column` children** in that row.
9. **Heading `title.innerContent` is plain text only** — no HTML tags. The heading tag (`h1`, `h2`, etc.) is controlled by `title.decoration.font.font.*.value.headingLevel`.
10. **Rich text content** (text modules, blurb body, testimonial content) must HTML-encode `<` as `\u003c` and `>` as `\u003e`.
11. **Never include `groupPreset`** — it references site-specific preset IDs that won't exist on other installations.
12. **Spacing is optional** — DIVI5 applies default padding when none is specified. Only add explicit spacing when overriding defaults.
13. **The `data` field value is a plain string** — not a nested object with `post_content`, `post_title`, etc.
14. **Rows CAN nest inside columns.** A column may contain both content modules AND nested rows (which then contain their own columns and modules). This is the correct way to build complex layouts like dashboard cards with inner sub-layouts.
15. **Cards are made with column or row decoration — NOT always with `divi/group`.** Apply `background`, `border.radius`, and `spacing.padding` directly to a `divi/column` or `divi/row` to create card-style containers. Use `divi/group` only when you need to card-wrap a *subset* of modules that share a column with other content.

## 2. Top-Level JSON Structure

```json
{
  "context": "et_builder",
  "data": {
    "11": "<!-- wp:divi/placeholder -->...PAGE_CONTENT...<!-- /wp:divi/placeholder -->"
  },
  "presets": null,
  "global_colors": [],
  "global_variables": [],
  "canvases": {"local": [], "global": []},
  "images": [],
  "thumbnails": []
}
```

### Field Descriptions

| Key | Required | Notes |
|-----|----------|-------|
| `context` | Yes | Always `"et_builder"` — validated server-side in `PortabilityPost.php` |
| `data` | Yes | Object with **one key** (any integer string, e.g. `"11"`). Value is the full page markup string. |
| `presets` | No | `null` for pages without saved presets |
| `global_colors` | No | Array of `[id, {color, status, label}]` tuples |
| `global_variables` | No | Array of variable objects |
| `canvases` | No | `{"local": [], "global": []}` for standard exports |
| `images` | No | `[]` |
| `thumbnails` | No | `[]` |

### Critical: `data` Value Format

The value of the `data["11"]` key must be a **plain string** containing the Gutenberg markup:

❌ **Wrong** (this is the `post_type` format — for layout libraries, not pages):
```json
"data": {
  "1": {
    "post_content": "<!-- wp:divi/... -->",
    "post_title": "My Page",
    "post_type": "page"
  }
}
```

✅ **Correct** (raw string for `et_builder` context):
```json
"data": {
  "11": "<!-- wp:divi/placeholder --><!-- wp:divi/section...<!-- /wp:divi/placeholder -->"
}
```

## 3. Import & Export — How It Works

When a JSON file is uploaded via DIVI's Import & Export UI:

1. WordPress calls the REST endpoint handled by `PortabilityController.php`
2. The controller validates `context === "et_builder"`
3. `PortabilityPost.php::import()` parses the file
4. `get_layout_content()` reads `data["11"]` as a string (the raw markup)
5. The markup is written to the current page's `post_content`
6. DIVI5 renders the Gutenberg blocks on next page load

The import will hard-fail with `"This file should not be imported in this context."` if the JSON `context` field does not match. Always use `"et_builder"` for page imports.

## 4. Module Reference

### 4.1 Layout Modules (Structural)

#### `divi/section`

Top-level container. Every page needs at least one. **Not self-closing.**

```json
{
  "module": {
    "meta": {
      "adminLabel": {"desktop": {"value": "Hero Section"}}
    },
    "decoration": {
      "background": {
        "desktop": {"value": {"color": "#0f172a"}}
      },
      "spacing": {
        "desktop": {"value": {"padding": {"top": "80px", "bottom": "80px"}}}
      }
    }
  },
  "builderVersion": "5.0.1"
}
```

#### `divi/row`

Horizontal container inside a section. Defines column arrangement via the flex system. **Not self-closing.**

Rows can also be **styled as cards** by adding `background`, `border`, and `spacing` directly to the row's `decoration` — this is the correct way to create a full-width card container that wraps other rows inside a column.

```json
{
  "module": {
    "advanced": {
      "flexColumnStructure": {
        "desktop": {"value": "equal-columns_3"}
      }
    },
    "decoration": {
      "layout": {
        "desktop": {"value": {"flexWrap": "nowrap"}},
        "phone": {"value": {"flexWrap": "wrap"}},
        "phoneWide": {"value": {"flexWrap": "wrap"}}
      },
      "sizing": {
        "desktop": {"value": {"maxWidth": "1280px"}}
      }
    }
  },
  "builderVersion": "5.0.1"
}
```

**Row-as-card pattern** (add background/border/spacing directly to the row):

```json
{
  "module": {
    "advanced": {
      "flexColumnStructure": {"desktop": {"value": "equal-columns_1"}}
    },
    "decoration": {
      "layout": {"desktop": {"value": {"flexWrap": "nowrap"}}},
      "background": {"desktop": {"value": {"color": "#1f1f1f"}}},
      "border": {"desktop": {"value": {"radius": {"topLeft": "2rem", "topRight": "2rem", "bottomLeft": "2rem", "bottomRight": "2rem", "sync": "on"}}}},
      "spacing": {"desktop": {"value": {"padding": {"top": "2rem", "bottom": "2rem", "syncVertical": "on", "syncHorizontal": "on", "left": "2rem", "right": "2rem"}}}}
    }
  },
  "builderVersion": "5.0.1"
}
```

**`flexColumnStructure` values:**

| Value | Columns |
|-------|---------|
| `"equal-columns_1"` | 1 column (full width) |
| `"equal-columns_2"` | 2 equal columns |
| `"equal-columns_3"` | 3 equal columns |
| `"equal-columns_4"` | 4 equal columns |
| `"offset-columns_9"` | Offset/asymmetric layout (use with explicit `flexType` on each child column) |

**`columnStructure` companion field** — optionally set alongside `flexColumnStructure` when using fractional splits:

```json
"advanced": {
  "flexColumnStructure": {"desktop": {"value": "equal-columns_2"}},
  "columnStructure": {"desktop": {"value": "1_2,1_2"}}
}
```

#### `divi/column`

A column inside a row. Contains content modules. **Not self-closing.**

Columns can be **styled as cards** by adding `background`, `border`, and `spacing` directly to the column's `decoration`. This is the **preferred pattern for card-style layouts** — put the card styling on the column itself, not on a wrapper group.

```json
{
  "module": {
    "decoration": {
      "sizing": {
        "desktop": {"value": {"flexType": "8_24"}},
        "phone": {"value": {"flexType": "24_24"}},
        "phoneWide": {"value": {"flexType": "24_24"}}
      }
    }
  },
  "builderVersion": "5.0.1"
}
```

**Column-as-card pattern** (background + border-radius + padding directly on the column):

```json
{
  "module": {
    "decoration": {
      "sizing": {
        "desktop": {"value": {"flexType": "12_24"}},
        "phone": {"value": {"flexType": "24_24"}},
        "phoneWide": {"value": {"flexType": "24_24"}}
      },
      "background": {"desktop": {"value": {"color": "#1f1f1f"}}},
      "border": {"desktop": {"value": {"radius": {"topLeft": "2rem", "topRight": "2rem", "bottomLeft": "2rem", "bottomRight": "2rem", "sync": "on"}}}},
      "spacing": {"desktop": {"value": {"padding": {"top": "2rem", "bottom": "2rem", "syncVertical": "on", "syncHorizontal": "on", "left": "2rem", "right": "2rem"}}}}
    }
  },
  "builderVersion": "5.0.1"
}
```

**Column layout control** — use `layout` inside `module.decoration` to control flex direction and alignment:

```json
"decoration": {
  "layout": {
    "desktop": {"value": {"flexDirection": "row"}}
  }
}
```

Use `"flexDirection": "row"` when placing multiple buttons side-by-side inside a column.
Use `"justifyContent": "center"` or `"alignItems": "center"` to align content inside the column.

**Column `advanced.type`** — can be set to signal fractional column type (used with `columnStructure` on the row):

```json
"advanced": {
  "type": {"desktop": {"value": "1_2"}}
}
```

**`flexType` cheat sheet (column width as fraction of 24):**

| flexType | Width | Use case |
|----------|-------|----------|
| `"24_24"` | 100% | Full width / single column |
| `"16_24"` | 66.7% | Two-thirds |
| `"12_24"` | 50% | Half |
| `"8_24"` | 33.3% | One-third |
| `"6_24"` | 25% | One-quarter |
| `"18_24"` | 75% | Three-quarters |
| `"4_24"` | 16.7% | Narrow sidebar |

#### `divi/group`

A generic `div` wrapper for grouping modules inside a column. **Not self-closing.**

Use `divi/group` when you need to visually card-wrap a **subset** of modules within a column that already has other content (e.g., two stat cards stacked in one sidebar column). For simpler card layouts with only one card per column, **prefer styling the column directly** (see column-as-card pattern above).

`divi/group` supports full decoration including `background`, `border`, `spacing`, and `layout`:

```json
{
  "module": {
    "decoration": {
      "background": {"desktop": {"value": {"color": "#1f1f1f"}}},
      "border": {"desktop": {"value": {"radius": {"topLeft": "2rem", "topRight": "2rem", "bottomLeft": "2rem", "bottomRight": "2rem", "sync": "on"}}}},
      "spacing": {"desktop": {"value": {"padding": {"top": "1rem", "bottom": "1rem", "syncVertical": "on", "syncHorizontal": "on", "left": "1rem", "right": "1rem"}}}},
      "layout": {"desktop": {"value": {"alignItems": "center"}}},
      "sizing": {"desktop": {"value": {"size": ["flexShrink", "flexGrow"]}}}
    }
  },
  "builderVersion": "5.0.1"
}
```

**`sizing.size` array values** for group flex behaviour:
- `"flexGrow"` — group expands to fill available space
- `"flexShrink"` — group shrinks when needed
- Use `["flexShrink", "flexGrow"]` to make a group fill its column height (full-height card)

**When to use `divi/group` vs column-as-card:**

| Situation | Use |
|-----------|-----|
| Single card per column | Column-as-card (background/border on column) |
| Multiple cards stacked in one column | `divi/group` for each card inside the column |
| Small inline UI element (badge, icon pill) | `divi/group` with tight padding |
| Email capture box alongside list content | `divi/group` wrapping form + text inside column |

#### `divi/group-carousel`

Carousel of `divi/group` slides. Each child `group` is one slide. **Not self-closing.**

### 4.2 Content Modules

#### `divi/heading`

Renders a standalone heading element. **Self-closing (`/-->`).**

> `"inlineEditor": "plainText"` — title value MUST be plain text. The HTML tag is set via `headingLevel`.

```json
{
  "title": {
    "innerContent": {
      "desktop": {"value": "Your Heading Text Here"}
    },
    "decoration": {
      "font": {
        "font": {
          "desktop": {
            "value": {
              "headingLevel": "h2",
              "size": "48px",
              "weight": "800",
              "color": "#FFFFFF",
              "textAlign": "center"
            }
          }
        }
      }
    }
  },
  "builderVersion": "5.0.1"
}
```

❌ **Wrong — never put HTML tags in heading value:**
```json
"innerContent": {"desktop": {"value": "<h2>My Title</h2>"}}
```
✅ **Correct — plain text only:**
```json
"innerContent": {"desktop": {"value": "My Title"}}
```

#### `divi/text`

Renders a block of HTML content (paragraphs, lists, etc.). **Self-closing.**

> Uses `"inlineEditor": "richtext"` — HTML content is expected and must be unicode-escaped.

```json
{
  "content": {
    "innerContent": {
      "desktop": {"value": "\u003cp\u003eYour paragraph text here.\u003c/p\u003e"}
    },
    "decoration": {
      "bodyFont": {
        "body": {
          "font": {
            "desktop": {
              "value": {
                "color": "#e2e8f0",
                "size": "18px",
                "textAlign": "center"
              }
            }
          }
        }
      }
    }
  },
  "builderVersion": "5.0.1"
}
```

#### `divi/button`

Renders a styled CTA button. **Self-closing.**

> Uses `linkUrl` and `linkTarget` — NOT `url`/`newWindow` (those are Divi 4 shortcode param names).
> 
> Use `"button.decoration.button.desktop.value.enable": "on"` to explicitly activate the button style engine (required when using the primary/CTA button style).

```json
{
  "button": {
    "innerContent": {
      "desktop": {
        "value": {
          "text": "Get Started",
          "linkUrl": "/contact",
          "linkTarget": "off"
        }
      }
    },
    "decoration": {
      "button": {
        "desktop": {"value": {"enable": "on"}}
      }
    }
  },
  "module": {
    "decoration": {
      "background": {"desktop": {"value": {"color": "#2563eb"}}},
      "border": {
        "desktop": {"value": {"radius": {"topLeft": "8px", "topRight": "8px", "bottomLeft": "8px", "bottomRight": "8px", "sync": "on"}}}
      },
      "spacing": {
        "desktop": {"value": {"padding": {"top": "18px", "bottom": "18px", "left": "48px", "right": "48px"}}}
      },
      "layout": {"desktop": {"value": {"justifyContent": "center"}}}
    }
  },
  "builderVersion": "5.0.1"
}
```

**Side-by-side buttons** — place two button modules inside a single column with `layout.flexDirection: "row"` set on the column:

```
Column (decoration.layout.desktop.value.flexDirection: "row")
  └── divi/button (Grab Freebies)
  └── divi/button (View Portfolio)
```

**`linkTarget`:** `"off"` = same tab, `"on"` = new tab.

## 5. Nesting Rules & Hierarchy

### 5.1 The Fundamental Hierarchy

```
placeholder wrapper
  └── Section
        └── Row
              └── Column
                    └── Module(s) or Group
                          └── Module(s) [if inside Group]
```

**Never place a module directly in a section or row.** Always follow: Section → Row → Column → Module.

### 5.2 Advanced Nesting — Rows Inside Columns (Nested Rows)

**Rows CAN nest inside columns.** This is the correct way to build complex dashboard-style or card layouts with inner sub-grids.

```
Section
  └── Row (outer — e.g. 2-col split)
        ├── Column A (16_24 — main content)
        │     ├── divi/heading
        │     ├── divi/text
        │     └── Row (inner — button row, equal-columns_1)
        │           └── Column (24_24, flexDirection: row)
        │                 ├── divi/button
        │                 └── divi/button
        └── Column B (8_24 — sidebar)
              ├── divi/group (stat card 1)
              │     ├── divi/icon
              │     ├── divi/heading
              │     └── divi/text
              └── divi/group (stat card 2)
                    ├── divi/heading
                    └── divi/text
```

When nesting rows inside columns, the inner row still needs `flexColumnStructure` and its own column children.

### 5.3 Card Patterns — Which Approach to Use

There are three valid ways to make a card in DIVI5. Choose based on layout complexity:

**Pattern A — Column-as-card** (simplest, most common):
Apply `background`, `border`, and `spacing` directly on the `divi/column`. Use when the whole column IS the card.

```
Row (equal-columns_2)
  ├── Column (12_24, background:#1f1f1f, border-radius:2rem, padding:2rem)  ← card
  │     ├── divi/heading
  │     ├── divi/text
  │     └── divi/button
  └── Column (12_24, background:#1f1f1f, border-radius:2rem, padding:2rem)  ← card
        ├── divi/heading
        └── divi/text
```

**Pattern B — Row-as-card** (for full-width card containers with inner layout):
Apply `background`, `border`, and `spacing` on the `divi/row`. Use when the card needs multiple inner rows for sub-layout (e.g. a contact section with nested title + form rows).

```
Section
  └── Row (equal-columns_1, background:#1f1f1f, border-radius:2rem, padding:2rem)  ← card
        └── Column (24_24)
              ├── Row (inner — heading row)
              │     └── Column → divi/heading + divi/text
              ├── Row (inner — form row)
              │     └── Column → divi/contact-form
              └── Row (inner — social links row)
                    └── Column → divi/text
```

**Pattern C — Group-as-card** (for multiple stacked cards within one column):
Use `divi/group` when ONE column needs to contain MULTIPLE separate cards stacked vertically.

```
Column B (8_24 — sidebar)
  ├── divi/group (card 1, background:#1f1f1f, border-radius:2rem, padding:1rem)
  │     ├── divi/icon
  │     ├── divi/heading
  │     └── divi/text
  └── divi/group (card 2, background:#1f1f1f, border-radius:2rem, padding:1rem)
        ├── divi/heading
        └── divi/text
```

### 5.4 Parent–Child Module Rules

| Parent | Valid Children |
|--------|----------------|
| `section` | `row`, `global-layout` |

| `column` | `group`, `group-carousel`, any content module, **nested `row`** |

## 6. Row Layout: The Flexbox System

**Divi applies `display: flex` automatically to rows** when `flexColumnStructure` is set. Never set `display: flex` yourself.

### 6.1 Required Row Attributes for Multi-Column Layout

Every multi-column row MUST have:

```json
{
  "module": {
    "advanced": {
      "flexColumnStructure": {
        "desktop": {"value": "equal-columns_3"}
      }
    },
    "decoration": {
      "layout": {
        "desktop": {"value": {"flexWrap": "nowrap"}},
        "phone": {"value": {"flexWrap": "wrap"}},
        "phoneWide": {"value": {"flexWrap": "wrap"}}
      }
    }
  },
  "builderVersion": "5.0.1"
}
```

### 6.2 Column Width Alignment Rule

The sum of all column `flexType` values in a row should equal 24.

| Layout | Column flexTypes |
|--------|-----------------|
| 1 column | `24_24` |
| 2 equal columns | `12_24` + `12_24` |
| 3 equal columns | `8_24` + `8_24` + `8_24` |
| 4 equal columns | `6_24` × 4 |
| 2/3 + 1/3 split | `16_24` + `8_24` |
| 3/4 + 1/4 split | `18_24` + `6_24` |

## 8. DIVI5 Variables / Design Tokens

### 8.1 Variable Reference Syntax

**Color variable:**
```
$variable({"type":"color","value":{"name":"gcid-ID","settings":{}}})$
```

**Font size / numeric variable:**
```
$variable({"type":"content","value":{"name":"gvid-ID","settings":{}}})$
```

**Global font family variable:**
```
$variable({"type":"content","value":{"name":"--et_global_heading_font","settings":{}}})$
$variable({"type":"content","value":{"name":"--et_global_body_font","settings":{}}})$
```

> **Critical:** Always include `"settings":{}` inside the variable reference. Omitting it breaks the reference.

### 8.2 Using Variables in Module Attributes

```json
"background": {
  "desktop": {
    "value": {
      "color": "$variable({\"type\":\"color\",\"value\":{\"name\":\"gcid-primary-color\",\"settings\":{}}})$"
    }
  }
}
```

Inside a JSON string context, the inner quotes must be escaped with `\"`.

### 8.3 Defining Variables in the Export File

**`global_colors` format:**
```json
"global_colors": [
  ["gcid-primary-color", {"color": "#0f172a", "status": "active", "label": "Primary Color"}],
  ["gcid-body-color", {"color": "#e2e8f0", "status": "active", "label": "Body Text Color"}]
]
```

**`global_variables` format (font sizes):**
```json
"global_variables": [
  {
    "id": "gvid-1ndmhhes58",
    "label": "Big Heading",
    "value": "clamp(2.5rem, 6vw, 5rem)",
```

## 10. Step-by-Step Build Workflow

### Step 1: Understand the Brief

Before writing any JSON, clarify:
- What sections are needed?
- How many columns per section?
- What are the brand colors and typography?
- Does the site use global design tokens?

### Step 2: Plan the Layout

Sketch the structure top-down:

```
[Section 1: Hero]            → 2-col row (text + image)
[Section 2: Stats]           → 3-col row (counters)
[Section 3: Features]        → 1-col header row + 3-col cards row
[Section 4: Testimonials]    → 1-col header + 2-col testimonial row
[Section 5: CTA]             → 1-col centered row
```

### Step 3: Select Modules

| Content type | Module |
|-------------|--------|
| Paragraph / rich text | `divi/text` |
| Standalone heading | `divi/heading` |
| Button only | `divi/button` |
| Image only | `divi/image` |
| Icon + title + text | `divi/blurb` |
| Title + body + button combined | `divi/cta` |
| Quote with attribution | `divi/testimonial` |
| Animated number | `divi/number-counter` |
| FAQ expandable | `divi/accordion` |
| Custom HTML/CSS | `divi/code` |
| Grouped modules | `divi/group` |
| Carousel of groups | `divi/group-carousel` |

### Step 4: Build Each Section

For each section, write in this order:
1. Open section tag with attrs
2. For each row:
   - Open row with `flexColumnStructure` + `layout.flexWrap`
   - For each column: open column with `flexType`, add modules, close column
   - Close row
3. Close section

### Step 5: Encode and Wrap

1. Build the complete markup string
2. Ensure HTML inside content values uses `\u003c` / `\u003e` encoding
3. Wrap entire content in `<!-- wp:divi/placeholder -->...<!-- /wp:divi/placeholder -->`
4. Place the string as the value of `data["11"]`
5. Build the root JSON object with all required keys

## 11. Programmatic JSON Generation (Python)

The most reliable way to generate valid DIVI5 JSON is using Python to build structured dicts and let `json.dumps()` handle all escaping.

```python
import json

BV = "5.0.1"

def section(attrs, children):
    a = json.dumps(attrs, ensure_ascii=False, separators=(',', ':'))
    return f'<!-- wp:divi/section {a} -->\r\n{children}<!-- /wp:divi/section -->\r\n'

def row(attrs, children):
    a = json.dumps(attrs, ensure_ascii=False, separators=(',', ':'))
    return f'<!-- wp:divi/row {a} -->\r\n{children}<!-- /wp:divi/row -->\r\n'

def column(attrs, children):
    a = json.dumps(attrs, ensure_ascii=False, separators=(',', ':'))
    return f'<!-- wp:divi/column {a} -->\r\n{children}<!-- /wp:divi/column -->\r\n'

def module(name, attrs):
    a = json.dumps(attrs, ensure_ascii=False, separators=(',', ':'))
    return f'<!-- wp:divi/{name} {a} /-->\r\n'

def make_row(col_count, children_list, max_width="1280px"):
    """Build a flex row with equal columns. children_list is a list of markup strings."""
    flex_map = {1: "24_24", 2: "12_24", 3: "8_24", 4: "6_24"}
    flex = flex_map.get(col_count, "24_24")

    row_attrs = {
        "module": {
            "advanced": {
                "flexColumnStructure": {"desktop": {"value": f"equal-columns_{col_count}"}}
            },
            "decoration": {
                "layout": {
                    "desktop": {"value": {"flexWrap": "nowrap"}},
                    "phone": {"value": {"flexWrap": "wrap"}},
                    "phoneWide": {"value": {"flexWrap": "wrap"}}
                },
                "sizing": {"desktop": {"value": {"maxWidth": max_width}}}
            }
        },
        "builderVersion": BV
    }

    cols = ""
    for child in children_list:
        col_attrs = {
            "module": {
                "decoration": {
                    "sizing": {
                        "desktop": {"value": {"flexType": flex}},
                        "phone": {"value": {"flexType": "24_24"}},
                        "phoneWide": {"value": {"flexType": "24_24"}}
                    }
                }
            },
            "builderVersion": BV
        }
        cols += column(col_attrs, child)

    return row(row_attrs, cols)

def build_export(page_content):
    """Wrap markup in the root export JSON structure."""
    full_content = f"<!-- wp:divi/placeholder -->{page_content}<!-- /wp:divi/placeholder -->"
    return {
        "context": "et_builder",
        "data": {"11": full_content},
        "presets": None,
        "global_colors": [],
        "global_variables": [],
        "canvases": {"local": [], "global": []},
        "images": [],
        "thumbnails": []
    }
```

## 12. Common Mistakes & Fixes

### 12.1 Wrong `data` field structure
❌ Nested object: `"data": {"1": {"post_content": "...", "post_title": "My Page"}}`
✅ Plain string: `"data": {"11": "<!-- wp:divi/placeholder -->...<!-- /wp:divi/placeholder -->"}`

### 12.2 Missing `builderVersion`
❌ Any module without `"builderVersion": "5.0.1"` will fail to render or import.
✅ Every module, row, column, and section must include it.

### 12.3 Row Flex Not Working
❌ Setting `display: flex` or `display: "flex"` on rows — this is ignored by Divi.
✅ Set `flexColumnStructure` in `module.advanced` and `flexWrap` in `module.decoration.layout`.

### 12.4 HTML Tags in Heading Content
❌ `"value": "<h2>My Title</h2>"` — tags will render literally as text.
✅ `"value": "My Title"` — plain text only. Set `headingLevel: "h2"` in the font decoration.

### 12.5 Self-Closing Syntax Error
❌ `<!-- wp:divi/text {...} -->` — this opens a container, never closes it.
✅ `<!-- wp:divi/text {...} /-->` — note the space before `/`.

### 12.6 Missing Closing Tags
Every container module needs a matching close tag: `<!-- /wp:divi/section -->`, `<!-- /wp:divi/row -->`, `<!-- /wp:divi/column -->`

### 12.7 `flexColumnStructure` Mismatch
If a row declares `equal-columns_3` but contains only 2 `divi/column` tags, Divi will render incorrectly.

### 12.8 Missing `<!-- wp:divi/placeholder -->` Wrapper
The entire page content must be wrapped. Missing this causes import failure.

### 12.9 Wrong Button Attribute Names
❌ Divi 4 names: `url`, `newWindow`
✅ Divi 5 names: `linkUrl`, `linkTarget`

### 12.10 Missing `settings: {}` in Variable References
❌ `$variable({"type":"color","value":{"name":"gcid-primary-color"}})$`
✅ `$variable({"type":"color","value":{"name":"gcid-primary-color","settings":{}}})$`

### 12.11 Blurb Title Format
❌ Plain string: `"innerContent": {"desktop": {"value": "Feature Title"}}`
✅ Object with `text` key: `"innerContent": {"desktop": {"value": {"text": "Feature Title"}}}`

### 12.12 Invalid Module Nesting
❌ Module directly in section (no row/column)
❌ Module directly in row (no column)
❌ Section nested inside a column (sections cannot nest)

### 12.13 Using `divi/group` for ALL cards (over-wrapping)
❌ Wrapping every card in a `divi/group` inside an already-dedicated column — creates unnecessary nesting.
✅ When a column IS the card, put `background`, `border`, and `spacing` directly on the `divi/column`. Only use `divi/group` when one column needs multiple separate card-styled sub-regions.

### 12.14 Side-by-side buttons without `flexDirection: row` on the column
❌ Placing two `divi/button` modules in a column without direction control — they will stack vertically.
✅ Set `"module.decoration.layout.desktop.value.flexDirection": "row"` on the column containing the buttons.

### 12.15 Contact form submit button missing from contact-form
❌ Adding a separate `divi/button` after `<!-- /wp:divi/contact-form -->` as the submit button.
✅ Set the submit button label and styles via the `button` top-level key on the `divi/contact-form` module itself:
```json
"button": {
  "decoration": {"button": {"desktop": {"value": {"enable": "on"}}}},
  "innerContent": {"desktop": {"value": {"text": "Submit"}}}
}
```

### 12.16 Spacing sync fields
When using sync for padding, include the sync flag alongside values:
✅ `"padding": {"top": "1rem", "bottom": "1rem", "syncVertical": "on", "syncHorizontal": "on", "left": "1rem", "right": "1rem"}`
The `syncVertical` and `syncHorizontal` fields are `"on"` or `"off"` strings, not booleans.

## 13. Validation Checklist

Before producing the final JSON, verify every item:

**Root structure:**
- [ ] `"context": "et_builder"` is present
- [ ] `"data"` has exactly one key (integer string)
- [ ] The `data` value is a plain string (not a nested object)

**Content string:**
- [ ] Starts with `<!-- wp:divi/placeholder -->`
- [ ] Ends with `<!-- /wp:divi/placeholder -->`
- [ ] Every section has at least one row
- [ ] Every row has at least one column
- [ ] No module is placed directly in a section or row

**Modules:**
- [ ] Every module has `"builderVersion": "5.0.1"`
- [ ] Every non-self-closing module has a matching close tag
- [ ] Every self-closing module ends with ` /-->` (space + slash)
- [ ] `flexColumnStructure` count matches actual column count per row
- [ ] Column `flexType` values are consistent with the layout intent
- [ ] Mobile breakpoints set for column widths (`phone`, `phoneWide`)
- [ ] Row `flexWrap: "wrap"` set on mobile breakpoints

**Card/layout patterns:**
- [ ] Cards use column-as-card, row-as-card, OR group-as-card — not a mix without reason
- [ ] `divi/group` is only used when multiple cards share a single column
- [ ] Side-by-side buttons have `flexDirection: "row"` on their parent column
- [ ] Contact form submit button is defined via `button` key on the `contact-form` module, not a separate `divi/button` after the form

**Content:**
- [ ] Heading `title.innerContent` values are plain text (no HTML tags)
- [ ] Rich text values in text/testimonial/blurb body modules are HTML-encoded (`\u003c`, `\u003e`)
- [ ] Blurb title uses `{"text": "..."}` object format
- [ ] Button uses `linkUrl` and `linkTarget` (not `url`/`newWindow`)
- [ ] Variable references include `"settings": {}`
- [ ] No `groupPreset` keys in any module
- [ ] Padding sync uses `"syncVertical": "on"` / `"syncHorizontal": "on"` string values

### Card Pattern Decision Tree

```
Need a card?
  ├── Whole column = one card?
  │     └── YES → Column-as-card (background/border/padding on column)
  ├── Full-width card with multiple inner rows?
  │     └── YES → Row-as-card (background/border/padding on row)
  └── Multiple cards stacked in one column?
        └── YES → divi/group per card inside the column
```

### Side-by-side Buttons Pattern

```
divi/column (decoration.layout.desktop.value.flexDirection: "row")
  ├── divi/button (Primary CTA)
  └── divi/button (Secondary CTA)
```
